## 一、项目全景 🌍

### 1.1 项目简介

**Aligo 智能旅行助手**是一个基于**大语言模型**的**多智能体旅行规划系统**，采用 **Plan-and-Execute** 架构设计。它就像一个「AI 旅行社」，能理解你的自然语言指令，自动完成行程规划、信息查询、偏好管理等任务。

### 1.2 技术栈总览

| 技术 | 用途 | 学习中需要关注的点 |
|------|------|------------------|
| 🐍 **Python 3** | 开发语言 | 异步编程、类型注解、模块化 |
| 🤖 **AgentScope 1.0.16** | 多智能体框架 | Agent 基类、消息传递机制 |
| 🧠 **豆包大模型** | 底层 LLM | Prompt 设计、意图识别 |
| 🗄️ **Milvus Lite** | 向量数据库 | RAG 知识库检索 |
| 📊 **BGE-small-zh-v1.5** | Embedding 模型 | 文本向量化 |
| 💾 **JSON 文件存储** | 长期记忆（当前实现） | 未来可升级为 PostgreSQL/Redis |
| 🌐 **DuckDuckGo** | 免费网络搜索 | 联网信息检索 |
| 🖥️ **Rich** | CLI 终端美化 | Python 终端 UI 开发 |

## 二、环境搭建 ⚙️

### 2.1 前提条件

- **Python 3.10+**（推荐 3.11）
- **pip** 包管理器
- **Git**

### 2.2 创建虚拟环境

```bash
# 进入项目目录
cd Lincan-Travel-Assistant-Agent

# 创建虚拟环境（隔离项目依赖）
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate     # Windows
```

> 💡 **为什么要用虚拟环境？**
> 每个 Python 项目依赖不同，虚拟环境可以隔离不同项目的包，避免版本冲突。

### 2.3 安装依赖

```bash
pip install -r requirements.txt
```

**依赖解读**：

| 包名 | 版本 | 作用 |
|------|------|------|
| `setuptools` | >=69.0.0,<82 | Python 包构建工具（Milvus Lite 依赖） |
| `agentscope` | 1.0.16 | **核心**：多智能体框架 |
| `requests` | 2.32.3 | HTTP 请求（LLM 调用） |
| `pymilvus[milvus_lite]` | 2.6.9 | **核心**：向量数据库 |
| `sentence-transformers` | 5.2.3 | **核心**：文本向量化 |
| `rich` | 13.9.4 | CLI 终端美化 |
| `ddgs` | 9.10.0 | DuckDuckGo 网络搜索 |

### 2.4 配置 API 密钥

编辑 `config.py`，填入你的豆包大模型（或兼容 API）的密钥：

```python
LLM_CONFIG = {
    "api_key": "your-api-key-here",   # 替换为你的API密钥
    "model_name": "doubao-seed-1-6-flash-250828",
    "base_url": "https://ark.cn-beijing.volces.com/api/v3",
    "temperature": 0.7,
    "max_tokens": 8192,
}
```

> ⚠️ **注意**：如果没有豆包 API 密钥，也可以用兼容 OpenAI API 的其他服务（如 DeepSeek、通义千问等），只需修改 `base_url` 和 `api_key`。

### 2.5 初始化知识库（可选）

```bash
python .claude/skills/ask-question/script/init_knowledge_base.py
```

### 2.6 验证安装

```bash
python cli.py
```

如果看到 Rich 风格的终端界面，说明环境搭建成功 ✅

## 四、关键设计思想 💡

### 4.1 Plan-and-Execute 架构

这是本项目的核心架构模式：

1. **Plan 阶段**：`IntentionAgent` 理解用户意图，制定执行计划
2. **Execute 阶段**：`OrchestrationAgent` 按计划调度各子 Agent 执行

> 🤔 **对比思考**：这就像你在做一个项目
> - Plan：先理解需求，制定方案（PM/架构师角色）
> - Execute：分工协作，各司其职（开发团队角色）

### 4.2 插件化设计（Skill Plugins）

每个子 Agent 都是一个独立的「Skill Plugin」：
- 独立开发、测试、部署
- 通过 `LazyAgentRegistry` 动态发现
- 懒加载：用到的才加载，启动仅需 3 秒

### 4.3 两层记忆架构

| 记忆类型 | 存储位置 | 生命周期 | 用途 |
|---------|---------|---------|------|
| 短期记忆 | 内存（滑动窗口） | 会话期间（10轮） | 上下文理解 |
| 长期记忆 | JSON 文件 | 永久 | 用户偏好、历史行程 |

### 4.4 稳定性保障

- **熔断器**：连续失败后自动熔断，保护下游服务
- **指数退避重试**：失败自动重试（最大3次）
- **健康检查**：输入 `health` 查看 LLM 服务状态
